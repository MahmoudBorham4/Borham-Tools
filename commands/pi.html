üí° How to Choose the Right Format?

Format selection depends on your file's column order. You can test multiple formats until you find the correct one that displays your data properly.

# Format Column Order Common Usage File Example
1 P X Y Z C Point Name, X Coordinate, Y Coordinate, Elevation, Code Surveying and Topography P1,100.50,200.75,15.20,TP
2 P Y X Z C Point Name, Y Coordinate, X Coordinate, Elevation, Code GIS Software and Google Maps P2,200.75,100.50,15.20,Building
3 X Y Z C X Coordinate, Y Coordinate, Elevation, Code (No Names) Laser Scanner Data 100.50,200.75,15.20,COLUMN
4 Y X Z C Y Coordinate, X Coordinate, Elevation, Code (No Names) Reverse Coordinate Systems 200.75,100.50,15.20,FENCE

‚úÖ Tips for Correct Selection

  • Open data file in Notepad or Excel to see column order
  • Try Format 1 first as it's most common in surveying
  • If points appear in wrong location, try Format 2 or 4 (swap X and Y)
  • If you don't have point names, use Format 3 or 4

Example 1

üèóÔ∏è Survey Project for Land Plot

Data:

  • 100 boundary points for plot
  • Point names: P1 to P100
  • X, Y, Z coordinates for each point
  • Descriptive codes: BOUNDARY

Implementation Steps:

  1. Compile data in CSV file
  2. Verify format: P,X,Y,Z,Code

Example 2

üì° Data from Total Station

Data:

  • 50 points for column axes
  • Data without names (numbers only)
  • X, Y, Z coordinates
  • Codes: COLUMN, BEAM, SLAB

Implementation Steps:

  2. File format: X,Y,Z,Code

üí° Advanced Tips and Guidelines

üîß File Preparation

  • Use Excel to organize data before export
  • Ensure comma is the column separator
  • Remove headers from CSV file
  • Verify numeric format is correct
  • Use Notepad++ to inspect text files
  • Save file as UTF-8 for Arabic support

‚ö° Performance Optimization

  • For 1000+ points: use selected options only
  • Disable point connection for large datasets
  • Use 2D points if elevations not important
  • Set text height appropriate to drawing scale
  • Split large data into multiple files
  • Use Undo if settings are wrong

üéØ Troubleshooting

  • If points don't appear: change format
  • If points in wrong location: swap X and Y
  • If errors appear: check commas in file
  • If texts too large/small: adjust text height
  • If some data missing: check import options
  • For Arabic characters: use UTF-8 encoding
