LUP Command

Automatically update point elevations from the nearest text - Smart connection between points and numbers

üìå What is the LUP Command?

The LUP (Level Update) command is a smart tool that searches for the closest text (Text/MText) for each point and extracts numbers from it to automatically update the point elevation (Z value). Instead of manually entering elevations for hundreds of points, the command connects points with the numbers written next to them in seconds.

üéØ Key Features

  • Automatic Linking: Automatically searches for the closest text for each point
  • Smart Number Extraction: Extracts numbers from complex texts
  • High Speed: Updates thousands of points in seconds
  • Isolation of Failed Points: Moves points without elevations to "Wrong_LVL" layer
  • Live Progress Bar: Real-time monitoring of updated points count

‚öôÔ∏è When to Use It?

  • When you have points and elevation numbers separate
  • After importing points from external files without Z values
  • When updating elevations of old points from new surveys
  • To connect survey points with written elevation readings
  • When you want to save hours of manual data entry

‚ö†Ô∏è Important Requirements

  • You must have POINTS in your drawing
  • You must have TEXT/MTEXT containing numbers
  • Texts must be close to points (Maximum distance: 5 units)
  • Numbers can be anywhere within the text

üí° Tips & Common Problem Solutions

‚úÖ For Best Results

  • Place texts close to points (less than 5 units)
  • Ensure texts contain valid numbers
  • Use Text or MText - both work with same efficiency
  • Number can be anywhere in the text

‚ö†Ô∏è Common Problems

  • Points in Wrong_LVL: Texts too far or contain no numbers
  • Incorrect numbers: Check if text contains only one number
  • Slow processing: Reduce unnecessary surrounding texts
  • Can't find texts: Check if texts are within 5 unit range

üîß Quick Solutions

  • Many failed points: Bring texts closer to points and rerun LUP
  • Inaccurate numbers: Clean texts from extra numbers
  • Wrong_LVL layer hidden: Open it from Layer Manager
  • After review: You can rerun LUP on corrected points

üî¥ Failed Points Isolation System

When LUP cannot find an elevation for a specific point, it automatically places it in the "Wrong_LVL" red layer. This makes it easy for you to find points that need review.

‚úÖ Successful Points

  • Remain in their original layers
  • Z updated automatically
  • Ready to use immediately

‚ùå Failed Points

  • Moved to "Wrong_LVL" layer
  • Appear in red color
  • Easy to find for review

üîÑ Recommended Review Steps:

  1. Open "Wrong_LVL" layer and turn it on
  2. Zoom to red points
  3. Check failure reason:
    • ‚Ä¢ Is there a nearby text?
    • ‚Ä¢ Does the text contain a number?
    • ‚Ä¢ Is distance greater than 5 units?
  4. Fix the problem (bring text closer or write correct number)
  5. Run LUP again on red points only
  6. After success, manually move points to correct layer
